This project demonstrates the implementation of secure access control in AWS using Identity and Access Management (IAM).

The system is designed following the principle of least privilege by creating users, groups, and roles with controlled access to cloud resources.

Key Features:
- Created IAM users, groups, and roles
- Implemented role-based access control (RBAC)
- Assigned permissions using AWS managed policies (S3, EC2)
- Designed full-access and read-only user groups
- Enforced least-privilege security model

Tech Stack:
- AWS IAM
- AWS S3
- AWS EC2

Outcome:
Developed a structured and secure access control system aligned with real-world cloud security practices.
